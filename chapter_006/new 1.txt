Оценить поведения срабатывания различных типов сборщиков мусора для программы из первого урока данной модуля.

Как вы считаете какой из сборщиков мусора подойдет наиболее оптимально для приложения заявок из второго модуля?

Какой тип сборщика будет оптимален для сервеного приложения?

При использовании Serial Collector (-XX:+UseSerialGC) объекты начинают удаляться лишь при их увеличении до 3000. 
При использовании дефолтного gc (скорее всего The Parallel GC - двухядерный процессор Intel® Core™ i5-3210M)
они начинали удаляться при 500.

При использовании The Parallel GC (-XX:+UseParallelGC) программа вела себя также как и при дефолтном gc. 
Изменение количества потоков (-XX:ParallelGCThreads=<desired number>) ни на что не повлияли.

При использовании Concurrent Mark Sweep (CMS) Collector (-XX:+UseConcMarkSweepGC) объекты начинают удаляться 
лишь при их увеличении до 3000. 

При использовании The G1 Garbage Collector (-XX:+UseConcMarkSweepGC) объекты начинают удаляться лишь при их 
увеличении до 3000. 

Наиболее оптимально для приложения заявок из второго модуля подойдёт Serial Collector для однопроцессорного 
компьютера или Parallel GC т. к. данное приложение не трубует быстрого отклика, для него более важна 
производительность.
